# Настройка мультиязычности в OnyxChips

## Что было сделано

1. **Настроена поддержка мультиязычности в Django:**
   - Добавлен `LocaleMiddleware` в settings.py
   - Настроены поддерживаемые языки: русский (ru), английский (en), испанский (es)
   - Добавлен контекст-процессор i18n

2. **Созданы файлы переводов:**
   - `locale/ru/LC_MESSAGES/django.po` и `django.mo`
   - `locale/en/LC_MESSAGES/django.po` и `django.mo`
   - `locale/es/LC_MESSAGES/django.po` и `django.mo`

3. **Добавлен API endpoint для смены языка:**
   - `/api/change-language/` - принимает POST запрос с параметром `language`

4. **Обновлен шаблон settings.html:**
   - Добавлены теги `{% trans %}` для перевода текста
   - Селектор языка теперь отправляет AJAX запрос при изменении
   - После смены языка страница автоматически перезагружается

## Как это работает

1. Пользователь выбирает язык в настройках
2. JavaScript отправляет POST запрос на `/api/change-language/`
3. Сервер сохраняет выбранный язык в сессии
4. Страница перезагружается с новым языком
5. Все тексты с тегами `{% trans %}` автоматически переводятся

## Как добавить переводы в другие шаблоны

1. Добавьте в начало шаблона: `{% load i18n %}`
2. Оберните текст в тег: `{% trans "Текст для перевода" %}`
3. Добавьте переводы в файлы `.po` в папке `locale/`
4. Запустите: `python compile_messages.py`

## Пример использования в шаблоне

```html
{% load i18n %}
<h1>{% trans "Добро пожаловать" %}</h1>
<p>{% trans "Это текст будет переведен" %}</p>
```

## Компиляция переводов

После изменения файлов `.po` запустите:

```bash
python compile_messages.py
```

Этот скрипт создан специально для работы без установки gettext.

## Поддерживаемые языки

- Русский (ru) - язык по умолчанию
- English (en)
- Español (es)

## URL с префиксами языков

Теперь все URL автоматически получают префикс языка:
- `/ru/` - русская версия
- `/en/` - английская версия
- `/es/` - испанская версия

Например:
- `/ru/settings/` - настройки на русском
- `/en/settings/` - настройки на английском
